# 🚀 Инструкция по локальному запуску и деплою на GitHub Pages

Этот документ поможет вам запустить сайт строительной компании **SREDA** на вашем компьютере (localhost) и опубликовать его бесплатно в интернете с помощью **GitHub Pages**.

---

## 💻 Часть 1. Запуск сайта на вашем компьютере (Localhost)

Для запуска проекта вам понадобится установленный **Node.js** (версии 18 или выше).

### Шаг 1. Скачайте проект и откройте терминал
Распакуйте архив с проектом в удобную папку на компьютере. Откройте терминал (или командную строку) и перейдите в эту папку:
```bash
cd путь/к/вашей/папке-с-проектом
```

### Шаг 2. Установите зависимости
В терминале выполните команду для скачивания всех необходимых библиотек (React, Tailwind CSS, Lucide Icons и др.):
```bash
npm install
```

### Шаг 3. Запустите сервер разработки
После успешной установки запустите локальный сервер:
```bash
npm run dev
```

В консоли появится ссылка. Обычно это:
👉 **`http://localhost:3000`** или **`http://localhost:5173`**

Откройте эту ссылку в браузере. Теперь любые изменения в коде будут мгновенно отображаться на странице!

---

## 🌐 Часть 2. Публикация на GitHub Pages

GitHub Pages бесплатно разместит ваш статический сайт. Так как сайт собран на сборщике **Vite**, есть две важные детали: настройка базового пути и сама сборка.

---

### 🛠️ Шаг 1. Настройка базового пути (Base Path)
Если ваш сайт будет доступен по адресу `https://ваш-логин.github.io/имя-репозитория/`, то Vite должен знать это имя репозитория, чтобы правильно загружать картинки и стили.

1. Откройте файл `vite.config.ts` на вашем компьютере.
2. Добавьте свойство `base: '/имя-вашего-репозитория/',` сразу внутри `defineConfig`:

**Пример изменения в `vite.config.ts`:**
```typescript
export default defineConfig(() => {
  return {
    base: '/sreda-remont/', // <-- ЗАМЕНИТЕ на точное имя вашего репозитория на GitHub!
    plugins: [react(), tailwindcss()],
    // ... остальной код
  };
});
```
*Примечание: Если ваш сайт будет привязан к персональному домену (например, `sreda-remont.ru`), то `base` указывать не нужно, оставьте его `'/'`.*

### 🚀 Вариант А. Быстрый деплой одной командой (Рекомендуемый)

Самый простой способ — использовать специальную утилиту `gh-pages`, которая сама соберет проект и отправит его в нужную ветку на GitHub.

1. Установите утилиту в проект:
   ```bash
   npm install -D gh-pages
   ```

2. Откройте файл `package.json` и добавьте две строчки в раздел `"scripts"`:
   ```json
   "scripts": {
     "dev": "vite --port=3000 --host=0.0.0.0",
     "build": "vite build",
     "preview": "vite preview",
     "lint": "tsc --noEmit",
     "predeploy": "npm run build",
     "deploy": "gh-pages -d dist"
   }
   ```

3. Инициализируйте ваш Git-репозиторий и свяжите его с GitHub (если ещё не сделано):
   ```bash
   git init
   git add .
   git commit -m "Initial commit for SREDA"
   git branch -M main
   git remote add origin https://github.com/ВАШ_ЛОГИН/имя-вашего-репозитория.git
   git push -u origin main
   ```

4. Запустите команду публикации:
   ```bash
   npm run deploy
   ```
   *Утилита автоматически создаст сборку в папку `dist` и отправит её в специальную ветку `gh-pages` на GitHub.*

5. Перейдите в настройки вашего репозитория на GitHub (**Settings** -> **Pages**) и убедитесь, что в качестве источника выбрана ветка **`gh-pages`** и папка **`/ (root)`**. Через 1-2 минуты ваш сайт будет доступен в сети!

### 🤖 Вариант Б. Автоматический деплой через GitHub Actions

Если вы хотите, чтобы сайт обновлялся сам при каждом обновлении кода в ветке `main`, вы можете настроить стандартный GitHub Workflow.

1. В корне проекта создайте структуру папок: `.github/workflows/`
2. Создайте файл `.github/workflows/deploy.yml` со следующим содержимым:

```yaml
name: Deploy Website

on:
  push:
    branches:
      - main # ветка, при пуше в которую запускается деплой

permissions:
  contents: write

jobs:
  build-and-deploy:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout Code
        uses: actions/checkout@v4

      - name: Set up Node.js
        uses: actions/setup-node@v4
        with:
          node-size: 20
          cache: 'npm'

      - name: Install Dependencies
        run: npm install

      - name: Build Project
        run: npm run build

      - name: Deploy to GitHub Pages
        uses: JamesIves/github-pages-deploy-action@v4
        with:
          folder: dist # папка, в которую собирается проект
          branch: gh-pages # ветка для публикации
```

3. Запушьте изменения в ветку `main` на GitHub. Сборка и деплой пройдут полностью автоматически!
